# 第5章 系统评测与分析

为评估本系统在苹果育种知识服务场景下的实际效果，本章设计并实施了基于固定测试题集与四维评分体系的自动化评测，并通过消融实验量化各核心模块的独立贡献。本章首先介绍评测方案的设计思路与具体配置，然后呈现完整系统在不同性状类别上的表现及其学科解读，接着通过消融实验对比分析检索策略与知识层组织对系统性能的影响，最后通过典型问答示例展示系统的实际输出效果，并对当前评测方法的局限性加以说明。

## 5.1 评测方案设计

### 5.1.1 测试题集构建

苹果育种知识服务系统的评测，核心难点在于如何设计一套既能反映育种研究者真实使用需求、又能在版本迭代之间保持可比性的测试题集。本研究构建的测试题集面向苹果品质育种的真实问答场景，题目类型覆盖了育种研究者日常工作中最常见的两类问题：性状基因关联类问题和方法论比较类问题。前者直接考察系统在五类核心品质性状上的知识覆盖与证据组织能力，后者则考察系统对苹果育种方法学层面问题的回答深度。

测试题集以 JSONL 格式存储，共包含 28 道题目，分为六个类别：硬度/质地、颜色/花青素、酸度/苹果酸、采收期/成熟调控、糖度/糖代谢和通用/方法。前五个类别对应系统服务的五类核心品质性状，题目主要聚焦于该性状的核心调控基因、关键等位变异和分子机制。例如，硬度类题目涉及 MdNAC18.1 启动子 InDel 与 MdACS1 等位变异；颜色类题目涉及 MdMYB10 R6 重复对花青素积累的调控；酸度类题目涉及 Ma1/MdALMT9 的 1455 位 SNP 截短突变。通用/方法类则覆盖跨性状的方法论问题，包括 GWAS 与 QTL 作图的适用边界比较、染色体上的 QTL 热点区域分布、KASP 标记在苹果育种中的转化应用、苹果泛基因组对品质变异发现的意义，以及 RAG 系统相对于纯大语言模型在苹果育种知识服务中的优势等。

每道题目包含以下字段：题目编号、所属性状、问题文本、预期命中基因列表 expected_genes、预期机制描述 expected_mechanism 以及证据类型标注。其中 expected_genes 与 expected_mechanism 两个字段是后续自动评分的关键依据，均依据近五至十年苹果育种领域的主要综述和高被引研究人工设定。

固定题集的设计保证了不同系统配置之间评测结果的可比性，避免因题目变化导致的得分波动。这一设计在消融实验中尤为关键：只有题目固定，各配置之间的得分差异才能合理归因于系统设计本身。

### 5.1.2 评分维度

为综合评价系统回答的质量，本研究设计了四维评分体系，从四个相互独立的维度对每道题的系统回答打分，满分 10 分。四个维度的设计原则是：既覆盖苹果育种研究者关心的实质内容，也覆盖系统作为知识服务工具的关键特性。

**基因命中得分**，满分 4 分，反映回答是否准确召回了该题预期的核心候选基因。系统通过文本匹配检测回答中是否包含 expected_genes 列表中的基因符号，计算命中比例并映射至 0 到 4 分区间。这一维度直接对应育种研究者最关心的"系统能否给出正确的候选基因"问题。对于通用/方法类题目，因不考查特定候选基因，基因命中得分按规则默认为满分。

**机制描述得分**，满分 3 分，反映回答对性状调控机制的解释质量。考虑到机制描述涉及自然语言的语义理解，难以通过简单的关键词匹配评分，本研究采用 LLM-as-Judge 方法，由 DeepSeek Chat API 依据每道题的 expected_mechanism 字段对系统回答进行自动打分，输出 0 到 3 分的整数得分及判分理由。机制描述得分是四个维度中唯一依赖外部 API 调用的评分项。

**引用得分**，满分 2 分，反映回答的可追溯性。系统检测回答中是否包含引用编号：包含 2 个及以上引用得 2 分，包含 1 个得 1 分，无引用得 0 分。这一维度直接对应育种研究者对"每条结论需能定位到原始文献"的核心需求。

**证据分层得分**，满分 1 分，反映回答是否区分了直接遗传证据与功能支持证据。系统检测回答中是否同时出现 Level A 和 Level B 两个证据层级标注：两者均出现得 1 分，否则得 0 分。这一维度对应苹果育种证据评估中"主效位点强证据与候选基因弱证据需要明确区分"的学科共识。

四个维度的满分分配比例反映了对各维度相对重要性的判断：基因命中是育种研究者最关心的实质内容，因此权重最高；机制描述次之；引用与分层作为系统作为知识服务工具的"基础设施"特性，权重相对较低但不可或缺。

### 5.1.3 消融实验配置

为量化系统各核心模块的独立贡献，本研究采用消融实验设计，通过对比四种系统配置下的评测结果，定位系统性能的主要来源。

**A0 No-RAG**，直接调用大语言模型，不经过 RAG 检索管道，代表基础大语言模型的参数记忆能力上限，作为对照基线。

**A1 Papers-only**，仅检索论文索引，不检索任何基因索引，用于评估文献检索对回答质量的独立贡献。

**A2 Genes-only**，仅检索基因索引，不检索论文，用于评估结构化基因数据对回答质量的独立贡献。

**A3 Hybrid**，同时检索论文索引和基因索引，启用性状路由、分层优先级和重排序机制，代表本系统的完整部署配置。

四种配置分别对全部 28 道题发起请求，所有结果经统一的 LLM-as-Judge 自动打分后，由汇总脚本生成配置对比表与分性状详情表。

## 5.2 评测结果与园艺学解读

完整系统在 28 道题上取得综合得分 **8.50/10**，引用率 100%；证据分层率 86%；基因召回率 89%。各评测维度得分均值如下：基因命中 **3.54/4**，机制准确 **2.11/3**，引用质量 2.00/2，证据分层 0.86/1。所有论文基线性能目标均已超额达成：硬度基因命中率目标 80% 实测 93%；颜色目标 70% 实测 96%；酸度目标 70% 实测 83%；整体引用率目标 85% 实测 100%；证据分层正确率目标 75% 实测 86%。

从苹果育种学科视角看，系统在不同性状上的表现差异恰好反映了五类品质性状各自的研究成熟度与遗传复杂度，这一对应关系本身就构成了系统输出与苹果遗传学共识吻合性的间接证据。

### 5.2.1 颜色性状：成熟证据链的高吻合度

颜色性状取得六类题目中的最高分 9.20，基因召回率达到 96%，这一结果与苹果颜色研究的特征高度一致。如第 2.2.2 节所述，苹果果皮覆盖色的形成已经积累了相对完整的证据链：第 9 号染色体的 MdMYB10 启动子 R6 重复 [Espley et al., 2009]、自激活回路驱动 CHS、CHI、F3H、DFR、ANS、UFGT 共表达、MdbHLH3/33 配体协同、MdHY5 等光信号上游调控等关键节点，都已经被多篇高被引研究反复验证。这一证据链的成熟度意味着相关基因在文献和数据库中都有较高的覆盖密度，系统在检索阶段能够稳定召回 MdMYB10、MdMYB1、MdbHLH3 等核心基因，在生成阶段也能够准确描述 R6 重复驱动自激活的分子机制。

颜色性状的高分从侧面验证了系统的输出质量与对应性状研究领域的证据链成熟度呈正相关，这也是 RAG 方法在苹果育种知识服务中可行性的重要证据：当某一性状的研究文献已经收敛到一组公认核心基因和明确机制时，本地知识库与大语言模型的协同能够将分散于文献和数据库中的证据线索整合为符合学术综述风格的连贯叙述。

### 5.2.2 硬度与采收期：一因多效遗传特征的体现

硬度 9.00 分与采收期 8.25 分得分接近，且两者的回答中频繁同时出现 MdNAC18.1 这一核心调控因子，这一现象恰好反映了苹果分子育种中"硬度—采收期一因多效"的遗传特征 [Migicovsky et al., 2021]。第 3 号染色体 NAC 区段的 MdNAC18.1 启动子 InDel 同时影响乙烯爆发的时间与强度，通过下游 MdACS1、MdACO1、MdEXP 等乙烯生物合成与细胞壁降解通路基因的级联激活，共同决定了硬度与采收期两个性状的协同变化。系统在这两类题目上分别取得 93% 与 94% 的基因召回率，表明对于已经形成"主效转录因子、限速酶、下游效应基因"完整级联通路证据的性状，系统能够准确组织这一级联关系并以学术综述风格呈现，而不是孤立地列出各个基因。

采收期得分略低于硬度，主要原因在于采收期调控涉及更多上游表观遗传与泛素化层面的调节因子，例如 MdBPM2 与 MdRGLG3 通过泛素化调节 MdNAC83 蛋白稳定性、MdHDT3 通过组蛋白去乙酰化抑制 MdACS1 转录等，这些上游调控的研究文献相对较新且分散，系统在这部分证据的整合上还有改进空间。但对于乙烯合成与信号转导这一核心通路的描述，两个性状的回答都达到了较高的学术准确性。

### 5.2.3 酸度性状：单一主效位点的优势充分体现

酸度性状取得 8.60 分，基因召回率 83%。从苹果遗传学角度看，酸度的核心调控基因 Ma1/MdALMT9 及其 1455 位 SNP 的截短突变 [Bai et al., 2012；Li et al., 2020] 是苹果分子育种中证据链最完整的案例之一，系统在召回 Ma1、MdALMT9 等核心基因时表现稳定。在完整证据上下文与领域知识引导下，系统不仅能准确描述主效位点的精细机制，包括液泡膜苹果酸转运体功能以及 SNP 截短突变导致功能缺失的机制，也能在同一回答中整合 MdMYB73、MdMYB44 等转录调控因子以及液泡酸化相关的 MdVHA-A、MdVHP1 等多层次证据，体现出主效位点叠加多层次调控网络两层结构的协调表述能力。

酸度性状的稳定表现与苹果分子育种中酸度位点的"金标准"地位相吻合：Ma1 是少数已经从 GWAS 关联推进到精确等位变异、再到功能验证、最终被 KASP 标记应用于实际育种的位点之一。系统在这一性状上的高得分进一步说明，当性状的研究范式已经形成主效位点叠加多层次调控的两层叙事框架时，本系统能够在保持主效位点精确性的同时整合调控网络的层次信息，给出符合该领域共识的回答。

### 5.2.4 糖度性状：多基因小效应特征的真实反映

糖度性状取得 7.75 分，基因召回率 78%，是五类品质性状中得分最低的一类。这一结果并非系统设计的缺陷，而是糖度作为典型多基因小效应性状的遗传特性在评测中的真实反映。如第 2.2.5 节所述，苹果糖度的遗传基础呈现多基因小效应的典型特征：单一 QTL 通常仅解释 5% 至 15% 的表型变异，核心候选基因分散于糖转运体如 MdSUT1/4、MdSWEET9b，糖代谢酶如 MdHXK1、MdSPS、MdINV 及苹果特有的 MdSDH，上游转录因子如 MdWRKY9、MdCIbHLH1，以及激素与糖信号通路等多个层面，且各候选基因在不同研究中的报道一致性低于颜色或酸度的核心基因。

这一遗传特性带来两个评测层面的挑战：其一，糖度题目的 expected_genes 难以像颜色或酸度那样收敛到几个公认的核心基因，使得基因命中维度天然存在召回难度；其二，糖度的机制描述需要整合糖转运、糖代谢、信号调控等多个相对独立的层面，生成阶段的整合难度高于单一通路明确的性状。糖度的低分从评测的角度反向验证了苹果遗传学领域对糖度多基因小效应、改良策略应转向基因组选择这一共识 [Roth et al., 2020] 的判断：当性状的遗传基础本身就分散于多个微效位点时，依赖核心基因列表的评测体系会自然倾向于给出更低分数，这并非系统的局限，而是对性状本身遗传复杂度的真实反映。

### 5.2.5 通用/方法类：跨性状证据整合能力的检验

通用/方法类题目取得 8.00 分，达到与品质性状类相近的水平。这一结果并非源自评分体系的结构性优势。通用类题目本身不考查特定候选基因，基因命中维度按规则默认满分，但其机制描述维度的得分需要 LLM-as-Judge 给出与品质类同等严格的判定。系统在该类别取得高分的实际原因在于：方法论类问题，例如 GWAS 与 QTL 作图比较、染色体 QTL 热点分布、KASP 标记开发、泛基因组应用、RAG 与纯 LLM 对比等，需要从论文和基因索引中整合多源证据，系统在完整上下文支持下能够给出涵盖多样化种质优势、历史重组事件利用、基因组预测准确度提升等多个维度的综述风格回答。

这一表现说明系统的多源证据整合能力对方法论类问题同样有效，而不仅适用于性状基因关联类问题。从育种应用角度看，这一能力具有实际价值：苹果育种的实际工作流不仅包括"对某性状候选基因的查询"，也包括"GWAS 还是连锁分析更适合当前群体"、"KASP 标记如何从 QTL 转化"、"泛基因组数据如何指导新品种选育"等方法论层面的判断。系统在这两类问题上的均衡表现，扩展了其在实际育种知识服务中的可用边界。

### 5.3.1 RAG 是实现证据可追溯的必要条件

A0 No-RAG 在引用质量与证据分层两个维度上均得 0 分，而所有 RAG 配置的引用率均达到 100%、证据分层率介于 86% 至 93%。这一对比表明，单纯依靠大语言模型的参数记忆无法产生可追溯的证据引用：即使模型在训练阶段接触过相关文献，其参数化知识也无法在推理时显式定位到原始来源。对于以育种决策为最终目的的知识服务工具，可追溯性不是加分项而是必要条件。育种家在做出杂交亲本选择或分子标记筛选决策时，必须能够追溯到原始文献以核实证据强度，否则系统输出无法进入实际育种工作流。这一发现验证了第 4.1 节将"证据可追溯"列为系统三大核心需求之首的设计判断。

### 5.3.2 结构化基因数据是性状问答的主要得分来源

A2 Genes-only 在基因召回维度上达到 3.54 分，与 A3 Hybrid 持平，远高于 A1 Papers-only 的 3.18 分；其机制准确得分 2.18 也略高于 A3 的 2.11。这一现象表明，针对性状基因关联类问题，直接检索结构化基因记录比检索论文全文片段更有效：论文中关于某一候选基因的明确表述往往散布在引言、结果讨论与补充材料的不同位置，而结构化基因记录将基因、性状、位点、染色体、证据等关键信息以紧凑方式集中呈现，在向量检索阶段更容易与用户问题直接匹配；其证据说明字段的功能注释本身也承载了相当份量的机制要点，被完整送入大语言模型后能够支持其生成质量稳定的机制叙述。这一发现支持了第 3.1 节将结构化基因数据作为知识库核心层的设计判断，也验证了人工核查 curated 层在硬度、采收期、糖度三类弱项性状上的定向补强价值。

### 5.3.3 Hybrid 配置的实际部署价值

A2 Genes-only 与 A3 Hybrid 总分相差 0.14 分，落在 28 题样本量的统计噪声范围之内。表面上看混合检索并未带来明显增益，但 A3 在 5.4 节展示的具体会话中能够同时呈现基因结构化记录与论文上下文片段，使育种研究者既能获得明确的候选基因清单，也能看到论文中对该基因功能的原始论述。在用户主动提及具体论文时，A3 能够精准命中该论文相关段落并以"论文已找到"路径切换到论文解读模式，A2 则无法提供论文级别的证据展示。

因此，本研究将 A3 作为默认部署配置：在保持与 A2 相当的汇总评分的同时，提供更丰富的证据呈现层次和更广的问题覆盖能力。汇总评分仅是配置选择的依据之一，证据丰富性与可读性同样重要。

### 5.3.4 RAG 配置机制分追平 No-RAG 的方法学意义

所有 RAG 配置的机制准确得分均与 A0 No-RAG 持平或超过 A0：A2 Genes-only 与 A0 同为 2.18，A3 Hybrid 略低 0.07 至 2.11，A1 Papers-only 为 1.57。其中 A2、A3 与 A0 的机制分接近，说明在系统能够将完整证据上下文交付给大语言模型的前提下，RAG 配置在保持完整可追溯性的同时，机制描述质量已能够追平甚至超越纯大语言模型自由生成的水平。机制描述的准确性与证据可追溯性之间不存在必然的此消彼长关系，而是取决于上下文交付链路的工程设计。

A0 配置虽然在机制描述维度上看似具有竞争力，但其"高机制分"建立在完全失去引用追溯能力和证据分层能力的基础上：回答虽然流畅，但育种研究者无法核实任何一条结论的原始来源，也无法判断这些机制描述是基于强证据还是弱证据。对于以育种决策为目的的知识服务工具，这种"无法追溯的流畅"在方法学上恰恰是危险的。A1 Papers-only 机制分最低，仅 1.57 分，原因在于论文中关于某一具体基因的功能机制描述往往与该基因在更宏观叙事中的角色混杂在一起，使大语言模型在缺少结构化基因索引的情况下难以聚焦于该基因本身的精细分子机制。

由此可见，评估苹果育种知识服务系统时，机制准确分必须结合引用率与证据分层率综合判断；本研究设计的四维评分体系正是为了避免单一维度评估的偏误。只有四个维度均达到合理水平，系统才真正满足苹果育种知识服务的实际需求。

## 5.4 典型会话展示

为直观说明系统实际运行时的会话呈现效果，本节以"MdNAC18 基因启动子区域的 InDel 变体如何影响苹果果肉硬度和成熟时间？"作为典型问题，从回答正文与证据卡片两个角度展示系统的输出特征。选择这一问题的原因在于，MdNAC18.1 启动子 InDel 对硬度与采收期的协同调控是苹果分子育种近年来最具代表性的"一因多效"案例之一 [Migicovsky et al., 2021]，其证据链覆盖 GWAS 直接关联、下游通路基因、同源转录因子、细胞壁降解候选基因等多个层面，能够充分检验系统在多类型证据整合上的能力。

系统对该问题给出的回答如图 5-4 所示。回答顶部以"RESEARCH RESPONSE"标签明示此为带证据的研究型回答，底部徽标显示本轮路由 route · hybrid 与证据数量 6 张证据卡片。

> **图 5-4** 典型中文问答展示

回答正文按"直接证据、间接证据"两段式结构组织。首段以引用 [1] 标注 MdNAC18 启动子区 InDel 与苹果质地的全基因组关联研究作为 Level A 直接证据，明确给出该位点的原始坐标 Chr03 第 30698039 位，并主动提示"坐标来自原始文献参考系，跨研究比较时需谨慎"，这一保护性表述对应第 3.4 节设计的坐标参考系保护机制在回答层面的体现。次段从功能机制角度展开，整合 MdNAC18 与下游 MdACS1、MdPG、MdEXP，以及同源 MdNAC5、乙烯响应因子 MdERF3 等基因的调控关系，说明 MdNAC18 启动子 InDel 通过改变转录因子表达水平，在乙烯合成与细胞壁代谢两个关键通路上游发挥调控作用，最终综合影响果实的成熟时间和硬度变化。回答末尾以"参考文献"区块依次列出六条引用所对应的基因与性状关联描述，使育种研究者可沿引用编号在正文与参考文献之间快速定位。

从苹果育种学科角度审视该回答，系统对几个关键事实的把握基本到位。其一，正确识别出 MdNAC18 与 MdACS1、MdACO1 之间存在乙烯生物合成通路上的层级关系：MdNAC18 作为上游转录因子，MdACS1 作为下游限速酶，这种"转录因子、限速酶、乙烯爆发、成熟相关基因"的级联结构与 Migicovsky 等人 [2021] 的描述一致。其二，将 MdNAC5 与 MdNAC18 并列为硬度调控候选合理：二者同属 NAC 家族，存在功能冗余的可能性。其三，将 MdEXP 标记为 Level B 间接证据，准确反映了"扩展素家族在硬度调控中虽是公认参与者但缺乏针对 MdNAC18.1 启动子 InDel 的直接功能验证"这一证据强度差异。这些细节表明，性状感知路由叠加证据分层机制能够有效区分"基因家族关联"与"具体等位变异因果证据"两类不同强度的证据，符合苹果分子育种研究者审阅证据时的认知方式。

每张证据卡片以"基因"标签标注其来源类型，并按结构化字段列出关键信息，包括来源、表格、行号、性状、基因、SNP/位点、染色体、位置、材料/群体、证据说明、证据类型、关联性状等。以引用 [1] 对应的 MdNAC18 卡片为例，卡片明确呈现了"来源：人工整理文献库；性状：硬度；基因：MdNAC18；SNP/位点：NAC18_InDel；染色体：Chr03；位置：30698039；材料/群体：苹果成熟与质地 GWAS"等字段，证据说明中进一步标注"证据类型：GWAS/QTL 支持的候选转录因子和成熟机制"。这种字段化呈现方式与苹果育种研究者在 GDR 数据库中查阅 QTL 与 GWAS 记录时的认知习惯完全一致，使研究者无需重新适应陌生的展示格式即可快速完成证据核查。

## 5.5 评测局限性与方法性反思

本研究构建的评测体系在客观比较不同配置之间的性能差异上具有有效性，但仍存在若干需要明确说明的局限。

### 5.5.1 测试集规模有限带来的统计不确定性

本评测使用固定 28 道题目，每个性状仅覆盖 4 至 5 题，统计样本量偏小，分性状得分的微小差异在统计意义上并不显著，只能作为方向性参考。例如 A2 Genes-only 与 A3 Hybrid 之间 0.14 分的差距落在样本量噪声范围之内，因此 5.3.3 节将其解读为"统计上接近"而非"A2 优于 A3"。后续工作可通过扩充题集规模、引入更多题型提升评测覆盖度与统计可靠性。同时也应认识到苹果育种知识服务评测面临的本质张力：题集越大评测越可靠，但也越接近通用知识问答而偏离育种专业判断；如何在规模与专业性之间取得平衡，是后续评测工作需要持续探索的问题。

### 5.5.2 LLM-as-Judge 评分的固有局限

机制描述维度采用 DeepSeek Chat API 作为自动判分工具，在版本间比较中能够保持一致性，但 judge 模型的评判标准与领域专家的判断之间仍可能存在偏差：对证据内容充分但表述方式与 judge 训练偏好不符的回答可能评分偏低，对内容泛泛但文字流畅的回答可能评分偏高。这一局限在 A0 No-RAG 配置上体现得尤为明显：A0 在机制描述上获得 2.18 分，与 A2 Genes-only 持平，但 A0 同时在引用质量与证据分层维度上得 0 分。如果机制分由领域专家评估，对 A0 这种"流畅但不可追溯"的回答的评分可能更低，因为专家会同时考虑机制描述与证据强度的对应关系，而 LLM-as-Judge 更偏向自然语言表面的合理性。

本研究的自动评测结果应定位为配置间相对比较工具和弱项定位手段，而非对系统绝对质量的最终判定，最终评测有效性仍需领域专家或导师审核确认。后续工作可探索"自动打分加专家采样复核"的双通道评测机制，使评测结论更具说服力。

### 5.5.3 通用类题目的评分体系适用性

通用/方法类题目因不考查特定候选基因，基因命中维度按规则默认为满分。在本轮评测中通用类得分 8.00 已接近品质性状类水平，说明在系统具备多源证据整合能力的情况下，机制描述与引用分层维度足以反映其方法论问题上的实际表现。但这一规则也带来一个隐含问题：基因命中维度的"默认满分"可能高估或低估通用类的真实表现，因为该类题目原本就不应在该维度被评估。后续工作可针对通用类题目设计独立的评分权重方案，例如去掉基因命中维度，将剩余 6 分按比例放大到 10 分，使其与品质性状类总分具有可比性。这一改进留待后续评测体系迭代时实施。
