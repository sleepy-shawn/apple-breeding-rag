# Archive Layout

`archive/` 只负责两类东西：学校材料，以及已经不再作为当前工作主线维护的历史文件。

## 当前结构

- `school-materials/`
  - 学校要求、任务书、开题材料、格式要求、参考模板和学校表格。
  - 这一层保持扁平，不再细分过多子目录，方便直接查找。

- `history/`
  - 历史性论文文件、中期材料、外文翻译、旧评测结果、旧报告、历史脚本和过程记录。
  - 这些文件保留追溯价值，但不再作为当前工作的主入口。

## 使用原则

- 当前正在写的论文、审阅清单和 Claude 交接文档不要继续放在 `archive/`，统一放到顶层 `thesis/`。
- `archive/` 中的内容默认视为“历史资料”或“学校资料”，不是运行路径。
- 如果某份资料还需要频繁修改，应移回 `thesis/` 或项目根目录相应功能区。
